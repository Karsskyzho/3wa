2 - JSX

Définition

JSX est une syntaxe facilitant l'écriture du code des composants, il sécurise également l'affichage des valeurs en échappant tous les caractères spéciaux (attaque XSS).

On utilise la casse camleCase en JSX.

content_copy
const el = <div className="main">Main</div>;

Exemple de JSX

En JSX on écrit:

content_copy
const el = (
  <h1 className="main">
    Hello, world!
  </h1>
);

En JS on écrit:

content_copy
const el = React.createElement(
  'h1',
  {className: 'main'},
  'Hello, world!'
);

La syntaxe JSX nécessite un compilateur (babel) que nous importons directement dans le fichier index.html pour compiler le code JSX en JS compréhensible par le navigateur.

Avec les navigateurs modernes le code JSX sera théoriquement correctement compilé.

ne pas oublier de charger le script JS avec l'attribut "text/babel"

Attributs

Ils s'écrivent soit dans des cotes simples ou doubles lorsqu'on a des chaîne de caractères comme valeur ou bien des accolades lorsqu'on a une expression en JS :

content_copy
const el1 = <h1 className="main">Main</h1>;
 
const el2 = <input required={inputIsRequired} value={inputVal} />;

Les balises enfants

Attention, on doit toujours mettre une div wrapper pour déclarer des éléments enfants.
Il faut se rappeler que la syntaxe JSX est du JS.
On verra que ces div parfois inutiles seront remplacées par du code React que l'on appelle des fragments.

Attaque XSS

La syntaxe JSX permet de se protéger des injections XSS, les caractères spéciaux seront échappés.

content_copy
const content = <script>alert('xss')</script>;
// Ceci est sans risque :
const el4 = <h1>{content}</h1>;

Parcourir un tableau JS

Pour parcourir un tableau JS dans le rendu d'un composant on ne peut utiliser que la méthode map sur celui-ci, ou bien créer une fonction de rendu qui effectue une boucle.
De plus chaque élément a besoin d'une clé unique pour être identifié par React dans le DOM.
La clé key est donc un mot réservé dans la librairie React, elle ne peut être récupérée :
